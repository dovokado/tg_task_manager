Before any changes: first describe **how you understood the request** (Phase 1) →
wait for "ок". If changes are involved, give **analysis + a plan** (Phase 2) →
wait for "ок" → only then execute.

**Why:** this removes rework caused by misread requests and keeps the user in control
of irreversible actions.

**How to apply:** phase depth depends on the tier (T0 — answer now; T1 — short
"understood → doing it"; T2 — full protocol). Fast-mode markers
(`fast`, `[fast]`, `швидко`, `безфазно`) skip the phases but keep safety checks.

Tier details are in the global `CLAUDE.md`.
